1. Project Overview

CodeLab Campus is an innovative web-based platform designed specifically for universities and colleges to enhance programming education. It bridges the gap between theoretical classroom learning and practical programming skills by providing a structured, hands-on environment for students to practice, test, and improve their coding abilities.

Built with Bolt.new for the World's Largest Hackathon, this platform aims to transform how programming is taught in educational institutions by giving faculty the tools to create customized coding exercises, track student progress, and conduct online coding assessments.

2. Problem Statement

Current Challenges in Programming Education

  • Universities often lack dedicated platforms for practical programming exercises
  • Faculty share programming problems as static PDFs or via learning management systems not designed for code
  • Students have limited options for integrated coding practice that aligns with their specific curriculum
  • No unified way to track student progress across programming assignments
  • Limited options for conducting online programming tests with automatic evaluation
  • Disconnect between theoretical lectures and hands-on programming practice

These challenges lead to a fragmented learning experience where students struggle to apply theoretical knowledge in practical scenarios, and faculty lack visibility into student progress and areas of difficulty.

3. Solution

CodeLab Campus solves these problems by providing a comprehensive platform with three integrated components:

Coding Lab Portal

A centralized repository where faculty can create and assign programming problems while students can practice, submit, and receive feedback.

Progress Tracking

Detailed analytics that show student performance by course, programming language, and assignment, enabling targeted support.

Online Test Module

Secure environment for timed programming assessments with optional proctoring features and automatic grading.

By integrating these components into a single, user-friendly platform, CodeLab Campus creates a continuous feedback loop between theoretical instruction and practical application, enhancing the overall learning experience.

5. System Architecture

CodeLab Campus is built on a modern, scalable architecture with clear separation of concerns:

Frontend Layer

User Interface & Interaction

Bolt.new / React

Backend Layer

Business Logic & API

Bolt.new REST API

Database Layer

Data Storage & Retrieval

PostgreSQL / MySQL

External Services

Code Execution & Auth

Judge0 API / Firebase Auth

Tech Stack Components

Component Technology Purpose
Frontend Bolt.new + React User interface, responsive design, interactive components
Backend Bolt.new REST API API endpoints, business logic, data processing
Database PostgreSQL / MySQL Relational data storage, complex queries, data integrity
Code Execution Judge0 API Secure code compilation and execution, test case validation
Authentication Firebase Auth / JWT User authentication, role-based access control
Deployment Vercel / Render Hosting, continuous deployment, scaling

System Integration

The architecture follows a service-oriented approach with clear API boundaries between components. The frontend communicates with the backend via REST APIs, while the backend interacts with both the database and external services like the code execution engine.

This separation allows for independent scaling of components based on demand, particularly important for the code execution service which may require additional resources during peak usage periods like online tests.

6. Database Design

The CodeLab Campus platform uses a relational database structure with the following key tables:

users

Field Type Description
id Integer (PK) Primary key
name String User's full name
email String (Unique) Email address, used for login
password String (Hashed) Password hash
role Enum student, faculty, or admin
college_id Integer (FK) Reference to colleges table

courses

Field Type Description
id Integer (PK) Primary key
name String Course name
description Text Course description
created_by Integer (FK) Reference to users table (faculty)

labs

Field Type Description
id Integer (PK) Primary key
title String Lab title
description Text Problem statement and instructions
language String Programming language (e.g., Java, Python)
score Integer Maximum points for this lab
course_id Integer (FK) Reference to courses table
created_by Integer (FK) Reference to users table (faculty)

Additional Core Tables

test_cases

  • id (PK)
  • lab_id (FK to labs)
  • input (test case input)
  • expected_output (expected result)
  • is_hidden (boolean, for test-only cases)

submissions

  • id (PK)
  • user_id (FK to users)
  • lab_id (FK to labs)
  • code (submitted solution)
  • output (execution result)
  • score (points awarded)
  • submitted_at (timestamp)

progress

  • id (PK)
  • user_id (FK to users)
  • course_id (FK to courses)
  • completed_labs_count (integer)
  • total_score (cumulative points)

tests

  • id (PK)
  • title (test name)
  • course_id (FK to courses)
  • start_time (datetime)
  • end_time (datetime)
  • is_proctored (boolean)
  • created_by (FK to users)

Relationship Diagram

Entity-Relationship Summary

  • One Faculty can create Many Courses
  • One Course contains Many Labs
  • One Lab has Many Test Cases
  • One Student makes Many Submissions
  • One Course can have Many Tests
  • One Test includes Many Labs via test_questions

Online Test Flow

  1. Faculty creates test by selecting labs, setting time limits and enabling proctoring if needed
  2. Students see upcoming test on their dashboard with scheduled date and time
  3. When test starts, students can access the problems and begin coding
  4. Timer counts down showing remaining time
  5. Students submit solutions for each problem
  6. System auto-evaluates submissions against test cases
  7. Test ends when time expires or student submits all solutions
  8. Results available to both student and faculty immediately

As a hackathon project, CodeLab Campus demonstrates innovation in educational technology, technical implementation excellence, and a clear vision for how AI-powered development tools can be used to solve real-world problems in higher education.
